# 이관 시작 프롬프트 (Light 모드) — 이거 하나면 됩니다

아래 "복사 시작 ~ 복사 끝" 사이를 복사해서 **빈칸 2개**를 채우고, 이관할 프로젝트를 연 AI(Claude Code / Codex)에 붙여넣는다.

AI는 **분석 → 스펙 초안까지 만들고 반드시 멈춘다.** 당신이 `02_Spec.md`를 검토하고 승인 체크를 한 뒤 "승인"이라고 말하면 그때 구현이 진행된다.

---

**══════ 복사 시작 ══════**

레거시 기능 이관을 시작한다. 아래 프로토콜을 순서대로 따르고, [멈춤 지점]에서는 반드시 멈춘다.

[입력]
- 기능명: `<여기에 기능명>`
- 레거시 경로: `<여기에 legacy repo 경로>`
- Kit 템플릿: 현재 프로젝트 또는 zip 경로에서 `templates/migration-docs-light/`를 찾아 사용. 못 찾으면 01_Analysis.md / 02_Spec.md / 03_Result.md 3개 파일을 같은 역할로 직접 생성.

[절대 규칙 — 전 단계 공통]
1. 레거시 경로는 read-only다. 어떤 파일도 수정하지 않는다.
2. 코드/쿼리에서 확인한 사실만 기록한다. 확인 못 한 동작은 "미확인"으로 남기고 절대 추측해서 구현하지 않는다.
3. `02_Spec.md`의 구현 승인 체크박스가 체크되기 전에는 production 코드를 한 줄도 작성하지 않는다.
4. 결제/인증/개인정보/여러 기능이 공유하는 코드를 건드려야 하면 멈추고 보고한다 (Full 모드 대상).
5. credential, 실서버 주소, 실데이터를 문서에 넣지 않는다.

[1단계 — 세팅]
- working tree가 clean인지 확인하고, clean일 때만 브랜치 생성: `feature/ai-migration-<기능명>`
- `docs/migration/<기능명>/`에 템플릿 3개를 복사(또는 생성)한다.
- 사용한 Kit 버전(commit hash 또는 zip 이름)을 01_Analysis.md 맨 위에 기록한다.

[2단계 — 분석]
- 레거시에서 추적: 진입점(URL/컨트롤러) → 호출 흐름 → 쿼리 조건(필터/삭제/상태) → 정렬 → 페이징 → 숨은 규칙(기본값, 예외 처리, 조용한 실패).
- Target 프로젝트에 같은 기능이 이미 있는지 확인한다.
- 결과를 `01_Analysis.md`에 기록한다. "미확인" 목록을 반드시 포함한다.

[3단계 — 스펙 초안]
- `02_Spec.md`에 이관 후 동작 계약 초안을 작성한다: API, 요청/응답, 빈 결과/에러, 정렬/페이징 기본값.
- 분석에서 나온 차이·모호한 지점을 "사람이 결정할 것" 체크리스트로 만든다. 임의로 결정하지 않는다.
- Target 프로젝트의 기존 코드 컨벤션(구조/네이밍/응답 형식)을 계약에 반영한다.
- "범위 제외"를 명시한다.

[멈춤 지점 — 반드시 멈춘다]
다음을 출력하고 사용자의 응답을 기다린다:
1. 분석 요약 3~5줄
2. "사람이 결정할 것" 목록
3. 안내: "docs/migration/<기능명>/02_Spec.md를 검토하세요. 결정 항목을 채우고 구현 승인 체크박스를 체크한 뒤 '승인'이라고 답하면 구현을 시작합니다."

[승인 후 — 구현] (사용자가 "승인"이라 답하고 + 02_Spec.md의 승인 체크박스가 실제로 체크된 것을 확인한 뒤에만)
- 승인된 계약 범위만 구현한다. "범위 제외"는 건드리지 않는다.
- 스펙에 없는 동작이 필요해지면 구현하지 말고 멈춰서 질문한다.
- 테스트를 함께 작성하고 실행한다. 실패는 그대로 보고한다 (숨기거나 assertion을 약화하지 않는다).
- `03_Result.md`에 기록: 변경 파일 / 실행한 테스트와 결과 / 검증 못 한 것 / 롤백 방법(브랜치 삭제).
- commit까지 하고 멈춘다. **push와 MR 생성은 사용자 지시가 있을 때만** 한다.

**══════ 복사 끝 ══════**

## 팀원용 요약 (당신이 하는 일 3가지)

1. 빈칸 2개 채워서 붙여넣기
2. AI가 멈추면 `02_Spec.md` 열어서 결정 항목 채우고 승인 체크 → "승인"
3. 결과 확인하고 push/MR 지시

## 문제가 생기면

- AI가 승인 전에 코드를 만들려 하면: "멈춰. 스펙 승인 전이야"라고 하면 된다 — 프로토콜 위반이다.
- 결정 항목이 뭘 물어보는 건지 모르겠으면: maintainer에게. 묻는 게 정상 동작이다.
