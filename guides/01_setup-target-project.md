# Set Up a Target Project

1. Review the Kit safety boundary.
2. Copy `templates/target-project/` into a public-safe test project.
3. Configure the private Validator dependency using an approved reference.
4. Keep `docs/migration/` as the validation root.
5. Keep `reports/` outside that root.

The exact Validator pin is confirmed in Phase 17. Commit hash pinning is the default recommendation.
