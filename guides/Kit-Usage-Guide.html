1. 준비물 확인

아래 4개가 준비되면 시작할 수 있습니다.

  1. Claude Code 또는 Codex가 설치되어 있어야 합니다. 터미널에서 확인: 
    claude --version   # Claude Code 사용자
codex --version    # Codex 사용자
    버전이 출력되면 OK. "command not found"가 나오면 각 도구의 공식 설치 안내를 먼저 따라주세요.
  2. git — git --version으로 확인. (macOS는 기본 설치되어 있습니다)
  3. 레거시 프로젝트와 신규 프로젝트가 내 컴퓨터에 클론되어 있어야 합니다. 
    git clone <레거시-repo-주소> ~/work/legacy-admin
git clone <신규-repo-주소> ~/work/new-admin-api
    이미 클론되어 있다면 그 경로를 그대로 쓰면 됩니다. 꼭 ~/work일 필요는 없습니다.
  4. Node.js 18 이상 (Validator와 npx 설치 방식에만 필요) — node --version으로 확인.
"경로"가 뭔가요? 이 문서에서 경로는 전부 내 컴퓨터 안의 폴더 위치입니다. ~는 내 홈 폴더(macOS: /Users/내이름)의 줄임말입니다. 폴더의 정확한 경로를 모르겠으면: Finder에서 폴더를 터미널 창에 드래그하면 경로가 자동 입력됩니다. 또는 그 폴더에서 pwd를 치면 현재 경로가 출력됩니다.

2. 설치 — Claude Code

터미널에서 2분이면 끝납니다.

  1. 터미널을 열고 신규 프로젝트로 이동

    cd ~/work/new-admin-api

  2. Claude Code 실행

    claude

    Claude Code 대화 화면이 열립니다. 이제부터는 이 대화 입력창에 입력합니다.

  3. 마켓플레이스 등록 (아래 한 줄을 입력창에 붙여넣고 Enter)

    /plugin marketplace add winkrj/ai-legacy-migration-framework-kit

    "marketplace added" 류의 성공 메시지가 나옵니다.

  4. 플러그인 설치

    /plugin install legacy-migration@legacy-migration-kit

  5. 설치 확인

    /plugin

    플러그인 관리 화면의 설치 목록에 legacy-migration이 보이면 성공입니다. 이제 입력창에 /legacy-migration:까지만 쳐도 커맨드 목록이 자동완성으로 뜹니다.

3. 설치 — Codex

플러그인 방식(권장)과 커스텀 프롬프트 방식 중 하나를 고르세요. 둘 다 설치해도 됩니다.

먼저 알아두세요 — 두 방법은 사용 방식이 다릅니다.
방법 A (플러그인)는 슬래시 커맨드를 만들지 않습니다. 설치 후 자연어로 요청하면 스킬이 워크플로우를 적용합니다.
방법 B (커스텀 프롬프트)를 설치해야 /migrate-start 같은 커맨드가 생깁니다. 둘을 같이 설치해도 됩니다.

방법 A. 플러그인 (권장)

  1. 터미널에서 마켓플레이스 등록

    codex plugin marketplace add winkrj/ai-legacy-migration-framework-kit

  2. Plugins 화면에서 설치

    Codex를 실행하고(codex) 입력창에 /plugins를 입력하면 플러그인 브라우저가 열립니다. 마켓플레이스 탭에서 Legacy Migration Kit을 찾아 Install을 선택하세요. (Codex 앱을 쓴다면: Plugins 화면 → 검색 → Add to Codex)

  3. 설치 확인

    새 스레드를 열고 이렇게 물어보세요:

    legacy-migration 스킬 설치돼 있어? 어떤 워크플로우인지 요약해줘

    분석 → 스펙 승인 → 구현 워크플로우를 요약해서 답하면 성공입니다.

방법 B. 커스텀 프롬프트

  1. 터미널에서 한 줄 실행

    npx --yes github:winkrj/ai-legacy-migration-framework-kit

    실행하면 아래처럼 출력됩니다:

    installed  /migrate-conventions  →  ~/.codex/prompts/migrate-conventions.md
installed  /migrate-full         →  ~/.codex/prompts/migrate-full.md
installed  /migrate-implement    →  ~/.codex/prompts/migrate-implement.md
installed  /migrate-start        →  ~/.codex/prompts/migrate-start.md
installed  /migrate-validate     →  ~/.codex/prompts/migrate-validate.md
완료. Codex 세션에서 사용: ...

  2. AGENTS.md 배치 (신규 프로젝트당 1회)

    kit의 안전 규칙을 상시 적용하려면 신규 프로젝트 루트에 AGENTS.md를 둡니다:

    curl -sL https://raw.githubusercontent.com/winkrj/ai-legacy-migration-framework-kit/main/agent/codex/AGENTS.md -o ~/work/new-admin-api/AGENTS.md

    이미 AGENTS.md가 있다면 내용을 병합하세요.

  3. 설치 확인

    Codex 입력창에 /migrate까지 치면 /migrate-start 등이 자동완성 목록에 뜹니다.

Codex에서는 승인 게이트 훅 동작이 아직 검증되지 않았습니다. 그래서 Codex 사용 시에는 분석은 한 세션에서, 구현은 새 세션에서 하는 것을 권장합니다. Claude Code는 훅이 차단해주므로 한 세션이어도 안전합니다.

4. 컨벤션 등록 (프로젝트당 1회, 권장)

이관된 코드가 팀 스타일(구조/네이밍/예외/응답 형식)을 따르게 하는 단계입니다. 건너뛰어도 이관은 되지만, 하면 결과물 품질이 확 올라갑니다.

  1. YOU신규 프로젝트에서 커맨드 실행

    팀 가이드(모범) 프로젝트가 ~/work/team-guide-project에 클론되어 있다고 가정하면:

    # Claude Code (~/work/new-admin-api에서 claude 실행 중)
/legacy-migration:conventions ~/work/team-guide-project

# Codex — 플러그인 설치했다면 (자연어)
컨벤션을 등록해줘. 참고 프로젝트는 ~/work/team-guide-project

# Codex — 커스텀 프롬프트(방법 B) 설치했다면
/migrate-conventions ~/work/team-guide-project

    참고 프로젝트가 없다면 인자 없이 실행하세요 — AI가 "직접 입력 / 현재 프로젝트에서 추출" 중 무엇으로 할지 물어봅니다. 규칙을 직접 알려주고 싶으면 실행 후 대화로 제시하면 됩니다 (예: "응답은 모두 ApiResponse<T>로 감싸고, 날짜는 ISO-8601 문자열로 반환해").

  2. AI초안 생성 후 멈춤

    AI가 반복 패턴과 반례를 수집해 ~/work/new-admin-api/docs/conventions/project-conventions.md를 Draft 상태로 만들고, 애매한 지점(스타일이 갈리는 곳)을 질문 목록으로 보여주고 멈춥니다.

  3. YOU검토하고 승인

    에디터로 docs/conventions/project-conventions.md를 열어 내용을 확인·수정하고, 문서 안의 Human Decision 섹션을 이렇게 바꿉니다:

    ## Human Decision   (바꾸기 전)
- Decision: Pending

## Human Decision   (바꾼 후)
- Decision: Approved
- Approved By: 홍길동
- Approved At: 2026-07-06

    Approved로 바뀐 순간부터 모든 이관 스펙·구현이 이 규칙을 의무적으로 따릅니다. Draft인 동안은 참고만 합니다.

5. 기능 이관하기 (전체 흐름)

여기가 본편입니다. notice-list(공지사항 목록) 기능을 이관하는 전 과정입니다.

  1. YOU이관 시작 커맨드 입력

    신규 프로젝트(~/work/new-admin-api)에서 AI를 실행한 상태에서, 기능명과 레거시 경로 2개를 넣어 입력합니다:

    # Claude Code
/legacy-migration:start notice-list ~/work/legacy-admin

# Codex — 플러그인 설치했다면 (자연어)
레거시 기능 이관을 분석 단계부터 시작해줘. 기능명 notice-list, 레거시 경로 ~/work/legacy-admin

# Codex — 커스텀 프롬프트(방법 B) 설치했다면
/migrate-start notice-list ~/work/legacy-admin

  2. AI분석하고 스펙 초안까지 만든 뒤 멈춤 (몇 분 소요)

    AI가 하는 일: ① feature/ai-migration-notice-list 브랜치 생성 ② 레거시 코드 추적(진입점 → 호출 흐름 → 쿼리 조건 → 정렬/페이징 → 숨은 규칙) ③ 문서 작성. 끝나면 신규 프로젝트에 이런 파일들이 생깁니다:

    ~/work/new-admin-api/
└── docs/migration/notice-list/
    ├── 01_Analysis.md   ← 레거시 분석 결과 + "미확인" 목록
    ├── 02_Spec.md       ← 이관 후 동작 계약 초안 ★ 당신이 볼 파일
    └── 03_Result.md     ← (아직 빈 템플릿, 구현 후 채워짐)

    그리고 AI는 분석 요약과 "사람이 결정할 것" 목록을 보여주고 멈춥니다. 레거시 폴더는 아무것도 수정되지 않습니다.

  3. YOU02_Spec.md 열어서 결정하고 승인

    에디터로 docs/migration/notice-list/02_Spec.md를 엽니다. 할 일은 딱 두 가지:

    ① "사람이 결정할 것" 체크리스트에 답 채우기 — 예를 들면:

    ## 사람이 결정할 것
- [ ] 삭제된 공지를 목록에 노출할지  ← 레거시는 노출했음
- [x] 삭제된 공지를 목록에 노출할지 → 노출하지 않는다 (신규 정책)

    ② 맨 아래 구현 승인 체크박스 체크 — [ ]를 [x]로 바꾸고 저장:

    - [ ] 위 계약대로 구현을 승인한다 (승인자: __________ / 날짜: __________)
- [x] 위 계약대로 구현을 승인한다 (승인자: 홍길동 / 날짜: 2026-07-06)
    이 체크박스가 게이트입니다. 체크되기 전에는 (Claude Code에서) 훅이 docs/migration/ 밖의 모든 파일 수정을 자동 차단합니다. AI가 규칙을 "깜빡"해도 물리적으로 구현이 불가능합니다.

  4. YOU구현 지시

    # Claude Code
/legacy-migration:implement notice-list

notice-list 스펙 승인했어. 승인된 범위만 구현 진행해줘

# Codex — 커스텀 프롬프트(방법 B)
/migrate-implement notice-list

  5. AI승인된 범위만 구현하고 멈춤

    AI가 하는 일: ① 승인 체크박스가 진짜 체크됐는지 확인 (안 됐으면 즉시 종료) ② 승인된 계약 범위만 구현 — Approved 컨벤션 준수, "범위 제외"는 안 건드림 ③ 테스트 작성·실행 (실패하면 실패한 대로 보고) ④ 03_Result.md에 변경 파일/테스트 결과/검증 못 한 것/롤백 방법 기록 ⑤ commit 하고 멈춤.

    구현 중 스펙에 없는 동작이 필요해지면 AI는 구현하지 않고 당신에게 질문합니다 — 그게 정상 동작입니다.

  6. YOU결과 확인 → push/MR 지시

    docs/migration/notice-list/03_Result.md를 열어 확인하고, 괜찮으면 AI에게:

    push하고 MR 만들어줘

    push와 MR은 항상 이렇게 사람이 지시할 때만 진행됩니다. 마음에 안 들면 브랜치를 지우면 깨끗하게 원상복구됩니다:

    git checkout main && git branch -D feature/ai-migration-notice-list

6. 고위험 기능은 Full 모드

다음 중 하나라도 해당하면 위의 Light 모드로 진행하지 마세요: 결제 · 인증/로그인 · 개인정보(PII) · 여러 기능이 공유하는 코드 · production 전환(cutover) 판단. AI도 분석 중 이런 코드를 만나면 스스로 멈추고 보고하게 되어 있습니다. 
# Claude Code
/legacy-migration:full payment-refund ~/work/legacy-admin

# Codex
/migrate-full payment-refund ~/work/legacy-admin

Full 모드는 문서 8개(Discover→Specify→Plan→Implement→Validate→Archive)와 OpenSpec을 사용하고, 각 단계 산출물마다 당신의 확인을 받은 뒤에만 다음 단계로 넘어갑니다. 나머지 사용법은 Light 모드와 같습니다.

7. 문서 검사 (Validator)

이관 문서가 형식을 갖췄는지(필수 섹션/필드, 상태 어휘, 민감정보, 구현 경계) 기계적으로 검사합니다. Full 모드에서는 필수, Light 모드에서는 선택.

# Claude Code
/legacy-migration:validate notice-list

# Codex — 플러그인 (자연어) / 커스텀 프롬프트(방법 B)
notice-list 이관 문서를 validator로 검사해줘
/migrate-validate notice-list

# 직접 터미널에서 돌리기 (설치 불필요, Node 18+)
npx --yes github:winkrj/legacy-migration-validator-cli \
  --root ./docs/migration/notice-list \
  --report ./reports/notice-list-validation-report.md

리포트는 reports/ 폴더에 생성됩니다. Validator 통과 = 문서 구조 OK일 뿐, 도메인 정확성 보장이 아닙니다 — 그건 당신의 승인과 테스트가 담당합니다.

8. 문제 해결

/legacy-migration: 커맨드가 자동완성에 안 떠요

① /plugin으로 legacy-migration이 설치·활성화(enabled) 상태인지 확인하세요. ② 마켓플레이스 추가를 건너뛰었다면 /plugin marketplace add winkrj/ai-legacy-migration-framework-kit부터 다시. ③ Claude Code를 재시작(Ctrl+C 후 claude)해보세요.

플러그인을 설치했는데 /migrate-conventions 커맨드가 없다고 나와요

정상입니다 — Codex 플러그인은 슬래시 커맨드를 만들지 않습니다. 플러그인은 스킬로 동작하므로 자연어로 요청하세요: "컨벤션을 등록해줘. 참고 프로젝트는 ~/work/team-guide-project". /migrate-* 커맨드를 쓰고 싶으면 방법 B를 추가로 설치하세요: npx --yes github:winkrj/ai-legacy-migration-framework-kit

Codex Plugins 화면에 Legacy Migration Kit이 안 보여요

터미널에서 codex plugin marketplace add winkrj/ai-legacy-migration-framework-kit을 먼저 실행했는지 확인하세요. 플러그인 브라우저에서 마켓플레이스 탭을 전환하면 legacy-migration-kit 마켓플레이스 아래에 나타납니다. Codex 버전이 오래됐다면 업데이트 후 다시 시도하세요.

npx 실행이 실패해요

① node --version이 18 이상인지 확인. ② 네트워크에서 github.com 접근이 가능한지 확인. ③ 회사 프록시 환경이면 zip으로 받아 수동 복사하세요: repo에서 Code → Download ZIP → 압축 해제 후 cp codex/prompts/*.md ~/.codex/prompts/

"BLOCKED by legacy-migration spec gate" 메시지가 떴어요

훅이 정상 동작한 것입니다. AI가 스펙 승인 전에 코드를 수정하려 했다는 뜻입니다. 02_Spec.md를 검토하고 승인 체크박스를 체크한 뒤 /legacy-migration:implement를 실행하면 풀립니다.

AI가 승인 전인데 코드를 만들려고 해요 (Codex)

"멈춰. 스펙 승인 전이야"라고 답하면 됩니다 — 프로토콜 위반이므로 AI가 멈춥니다. 이런 일을 줄이려면 분석 세션과 구현 세션을 분리하고, AGENTS.md를 프로젝트 루트에 두세요.

"사람이 결정할 것" 항목이 무슨 뜻인지 모르겠어요

AI에게 그 항목을 그대로 물어보세요 ("삭제 노출 여부가 왜 결정이 필요한지 레거시 근거를 보여줘"). 그래도 애매하면 팀 리드/도메인 담당자에게 물어보는 게 맞습니다. 임의로 아무거나 체크하고 승인하는 것이 가장 위험합니다.

테스트가 실패한 채로 끝났어요

정상 동작입니다 — kit은 실패를 숨기거나 assertion을 약화하는 것을 금지합니다. 03_Result.md의 실패 내역을 보고 스펙 문제인지 구현 문제인지 판단한 뒤 재지시하세요.
