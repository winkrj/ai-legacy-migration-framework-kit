# migrate-implement — 승인된 스펙 범위만 구현

인자: `<기능명>` — 비어 있으면 현재 브랜치 `feature/ai-migration-<기능명>`에서 추출하고, 그래도 없으면 사용자에게 물어본다.

[승인 확인 — 통과 못 하면 즉시 종료]
1. `docs/migration/<기능명>/02_Spec.md`를 읽는다.
2. "구현 승인" 체크박스가 실제로 체크(`- [x]`)되어 있는지 확인한다.
3. 체크되어 있지 않으면 구현하지 않는다. "02_Spec.md의 구현 승인 체크박스가 아직 체크되지 않았습니다. 검토 후 체크하고 다시 실행해주세요."를 출력하고 종료한다.
4. "사람이 결정할 것" 중 답이 비어 있는 항목이 있으면 목록을 보여주고 종료한다.

[구현 규칙]
- 승인된 계약 범위만 구현한다. "범위 제외"는 건드리지 않는다.
- 스펙에 없는 동작이 필요해지면 구현하지 말고 멈춰서 질문한다.
- Target의 승인된 컨벤션을 그대로 따른다. 컨벤션과 스펙이 충돌하면 멈추고 질문한다.
- 레거시 경로는 여전히 read-only다.

[검증]
- 테스트를 함께 작성하고 실행한다. 실패는 그대로 보고한다 — 숨기거나 assertion을 약화하지 않는다.
- `03_Result.md`에 기록: 변경 파일 / 실행한 테스트와 결과 / 검증 못 한 것 / 롤백 방법(브랜치 삭제).

[종료]
- commit까지 하고 멈춘다. push와 MR 생성은 사용자 지시가 있을 때만 한다.
- 변경 요약, 검증 결과, 다음 권장 액션 1개를 출력한다.
